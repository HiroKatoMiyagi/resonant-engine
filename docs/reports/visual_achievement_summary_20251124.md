## 🎓 教訓

### DO（実践すべきこと）

```
✅ 実測でのみ評価する
   → ドキュメントの有無 ≠ 実装の有無

✅ テストコードを信頼する
   → pytest.skip()がない = 動作している

✅ 実装状況は実行で確認
   → ファイル存在 + テスト合格 = 実装完了

✅ 保守的評価は適度に
   → 過度な悲観は不要な不安を生む
```

### DON'T（避けるべきこと）

```
❌ 推測だけで評価しない
   → "おそらく未実装" は危険

❌ ドキュメント依存の評価
   → ドキュメント不足 ≠ 実装不足

❌ ファイル名だけで判断
   → detector.py存在 → でも中身確認せず

❌ テストスキップを許容
   → スキップは問題を隠蔽する
```
